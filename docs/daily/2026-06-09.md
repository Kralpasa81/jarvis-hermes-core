## Safe architecture note

OCR workflows should split into two clear lanes:

- **private runtime lane** — scans, extracted text, original filenames, personal IDs, and local folders stay outside the repository;
- **public planning lane** — generic checklist steps, placeholder labels, and redacted status shapes can live in docs or examples;
- **approval lane** — sending, filing, deleting, uploading, or sharing document results requires explicit human confirmation.

This keeps the repository useful for designing the workflow while avoiding accidental disclosure of personal documents or infrastructure paths.
