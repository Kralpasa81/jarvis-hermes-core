# Daily Jarvis / Hermes Lab Note — 2026-06-18

## Today's focus

Define a small public-safe **module handoff checklist** for future Jarvis / Hermes features before they move from idea notes into runnable scripts or private automations.

The goal is to make each module easier to review: what it reads, what it may output, what requires approval, and what must remain outside the public repository.

## Safe architecture note

A Jarvis / Hermes module should have a clear handoff boundary before implementation:

1. **Public contract** — describe the module with generic inputs, generic outputs, and no account-specific identifiers.
2. **Private adapter** — keep tokens, local endpoints, device IDs, OAuth files, and personal documents in private runtime configuration.
3. **Preview mode** — provide a harmless dry-run path that prints sample decisions without sending messages, controlling devices, or publishing data.
4. **Approval checkpoint** — require confirmation before external posts, destructive actions, payments, device control, or private-data processing.
5. **Audit-friendly note** — document what was intentionally left out of the public repo so future contributors do not mistake placeholders for missing work.

This keeps the public project useful while preventing accidental leakage of private automation details.

## Small actionable task

For the next new example or workflow note, add a short handoff block that answers:

- What synthetic input can be shared publicly?
- What output is safe to show in README/docs examples?
- Which action level is allowed by default: read-only, preview-only, approval-required, or private-runtime-only?
- Which secrets or personal data must never be committed?
- How can the module be tested without real services?

## Future module idea

Create a `module_handoff_checklist.py` helper that prints a Markdown checklist for a proposed module. It should accept a module name and action level, then generate safe sections for public contract, private adapter, preview behavior, approval rules, and secret boundaries.

## Security / secret reminder

Do not commit API keys, tokens, OAuth credentials, webhook URLs, home-network addresses, device identifiers, personal files, chat logs, payment data, screenshots with private state, or real automation outputs that reveal sensitive context.
