# Daily Jarvis / Hermes Lab Note — 2026-06-08

## Today's focus

Add one public-safe guardrail that helps future Jarvis / Hermes examples stay clean before they become real integrations.

The practical goal is to make template validation part of the command-center habit: before a config shape is copied into a private setup, confirm that the public version contains only placeholders and safe labels.

## Safe architecture note

Public examples should describe **shape**, not **identity**:

- use placeholder module names instead of real device names;
- store environment labels like `public-demo`, not hostnames or home-network details;
- keep risky actions behind explicit `approval-required` labels;
- reject fields that look like tokens, passwords, OAuth secrets, or API keys.

This keeps the repository useful for design work without turning it into a leak surface.

## Small task added

Added `examples/config_template_validator.py`, a dependency-free Python helper that validates a small Jarvis / Hermes config template.

It can run with its embedded safe example, or it can check a JSON file passed with `--file`. The validator fails closed on secret-like key names and on module risk levels outside the documented safe set.
