# Daily Jarvis / Hermes Lab Note — 2026-06-17

## Today's focus

Add a small public-safe **workflow readiness gate** helper so future Jarvis / Hermes ideas can be labelled before they become real automations.

The goal is to keep the repo useful for planning while making the boundary clear between public examples, private runtime configuration, and actions that need explicit approval.

## Safe architecture note

Every workflow should publish only a safe planning surface in this repository:

1. **Generic name** — describe the workflow without real account names, device names, chat IDs, or private file paths.
2. **Synthetic input** — use placeholders or mock data instead of live personal data.
3. **Non-executing preview** — show what would happen without sending messages, calling APIs, changing devices, or writing private state.
4. **Approval rule** — mark whether the workflow is read-only, notification-only, external-publish, device-control, or high-impact.
5. **Private runtime boundary** — keep tokens, OAuth files, local endpoints, and real documents outside the public repo.

This keeps public documentation honest: a workflow can be planned, previewed, or blocked without pretending that private integrations are already complete.

## Small actionable task

Use `examples/workflow_readiness_gate.py` to classify one planned workflow as:

- `draft` when it still needs safe sample input or preview output;
- `preview_ready` when it has synthetic input and a non-executing preview;
- `approval_required` when it could publish externally, control devices, delete data, or affect money;
- `private_runtime_only` when it depends on credentials, private endpoints, personal files, or real device state;
- `blocked_for_public_repo` when the draft contains secrets or sensitive content.

## Future module idea

Create a public-safe workflow registry that stores each module's readiness label, approval rule, preview command, and private-runtime boundary. A dashboard could then show which ideas are safe demos and which must stay private.

## Security / secret reminder

Never commit API keys, OAuth tokens, webhook URLs, chat logs, exact home-network addresses, device credentials, personal documents, payment data, or screenshots that reveal private automation state.
