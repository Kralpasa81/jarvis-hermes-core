# Daily Jarvis / Hermes Lab Note — 2026-06-16

## Today's focus

Define a small public-safe pattern for a **workflow readiness gate** before any Jarvis / Hermes automation is connected to private accounts, home devices, or live services.

The goal is to make each future workflow prove three things first: it has safe inputs, it can preview the action, and it has a clear approval rule.

## Safe architecture note

A workflow should move through readiness gates instead of jumping directly from idea to live automation:

1. **Describe** — write the workflow purpose using generic labels and synthetic examples only.
2. **Validate** — check that required fields are present without reading secrets or local private config.
3. **Preview** — show the planned result without sending messages, changing device state, or calling external APIs.
4. **Approve** — require explicit confirmation for publishing, device control, destructive actions, or financial impact.
5. **Run privately** — keep real credentials, endpoints, personal documents, and device names outside the public repository.

This keeps public planning useful while preserving a hard boundary between documentation, demo code, and private runtime operation.

## Small actionable task

Add a future checklist or helper that labels workflow status as one of:

- `draft` — public idea only;
- `preview_ready` — safe sample input and output exist;
- `approval_required` — action may affect external systems;
- `private_runtime_only` — implementation depends on secrets, personal data, or device configuration;
- `blocked_for_public_repo` — content would expose private data or unsupported claims.

## Future module idea

Create a lightweight workflow registry that lists planned modules, their readiness gate, approval rule, and safe preview command. The dashboard could show this registry before any real integrations are enabled.

## Security / secret reminder

Never commit API keys, OAuth tokens, webhook URLs, real chat logs, exact home-network addresses, device credentials, personal documents, payment data, or screenshots that reveal private automation state.
