## Safe architecture note

A module handoff checklist should sit between planning docs and real integrations. It is useful because it makes hidden assumptions visible before code touches live services.

Recommended boundary:

1. **Public module contract** — name, goal, generic inputs, and generic outputs only.
2. **Action level** — mark the workflow as read-only, preview-only, approval-required, private-runtime-only, or blocked.
3. **Synthetic test path** — verify behavior with mock labels and sample payloads instead of real accounts, files, or devices.
4. **Private adapter warning** — keep credentials, OAuth state, local URLs, chat IDs, device IDs, and personal documents outside this repository.
5. **Rollback / manual path** — document how the user can stop or bypass the workflow if the private implementation fails.

This separation allows public examples to remain useful without pretending that private infrastructure exists in the repo.
