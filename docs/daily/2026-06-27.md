## Safe architecture note

A useful assistant module should be designed in layers:

1. **Public contract** — a README, lab note, or example that explains what the module is allowed to do using synthetic data only.
2. **Preview adapter** — a dry-run path that shows the action summary without calling external services or exposing private payloads.
3. **Private runtime adapter** — the only layer that may read local secrets, personal files, real device identifiers, or account-specific settings.
4. **Approval gate** — required for actions that post messages, change device state, delete data, make purchases, book services, or contact other people.

Keeping these layers separate makes the public repository useful without turning it into a container for private configuration.

## Security / secret reminder

Do not commit API keys, tokens, OAuth files, webhook URLs, private chat logs, home-network IP addresses, real device names, personal documents, payment details, or production claims that have not been verified. Public notes should describe patterns and boundaries, not private runtime facts.
