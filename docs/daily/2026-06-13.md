The goal is to make automation feel helpful without turning public examples or private adapters into silent control paths.

## Safe architecture note

A reliable command center benefits from a simple action lifecycle:

1. **Intent captured** — the assistant records the requested task in public-safe language.
2. **Risk classified** — the module labels the task as read-only, preview-only, approval-required, or blocked.
3. **Preview prepared** — the user can inspect what would happen before any real action runs.
4. **Execution logged safely** — results are summarized without tokens, private endpoints, file paths, or personal data.

Keeping these stages separate helps future modules share the same safety model while allowing each integration to hide its private runtime details.

## Security / secret reminder
