# Daily Jarvis / Hermes Lab Note — 2026-07-06

## Today's focus

Close the loop on the module-registry idea from earlier this week. Instead of only listing modules, add a small helper that reads registry metadata and flags entries whose `next_review` date is overdue or coming up soon. This turns the registry from a static list into something that can actually prompt a review.

## Safe architecture note

A registry is only useful if stale entries get noticed. The safest way to do that publicly is to check dates and status fields only — never private runtime state. `module_health_review.py` reads `slug`, `title`, `status`, `approval_level`, and `next_review` from a JSON registry (either a file, stdin, or a small embedded demo) and reports `ok`, `due-soon`, or `overdue` per entry. It never touches account IDs, tokens, device addresses, or real service endpoints, and it works from a fresh clone with zero external dependencies.

## Small actionable task

Pipe `module_registry_builder.py --format json` into `module_health_review.py` to see which planned modules are coming up for review soon. This is a good first check before starting any new module work, so review dates do not silently drift.

## Future module idea

A tiny "registry diff" helper that compares two JSON registry snapshots (e.g. this week vs. last week) and reports which modules changed `status` or `approval_level`. This would make it easy to see, at a glance, whether a module actually moved from `planned` to `prototype` to `preview-ready`, instead of relying on memory or scrolling through daily notes.

## Security / secret reminder

Do not commit API keys, tokens, OAuth files, webhook URLs, real IP addresses, device credentials, private chat logs, personal documents, payment data, or claims that an integration is production-ready before it has actually been built and verified. `module_health_review.py` only accepts synthetic JSON metadata — it does not read environment variables, network resources, or any file outside the path explicitly passed to `--registry`.
