# Daily Jarvis / Hermes Lab Note — 2026-06-20

## Today's focus

Add a tiny public-safe helper for reviewing example payloads before they are copied into docs, dashboard mockups, or assistant workflow notes.

The goal is not to scan private machines or prove full compliance. It is a lightweight reminder that even harmless-looking JSON can leak identifiers, credentials, webhook URLs, or local environment details if copied directly from a real runtime.

## Safe architecture note

Public examples should pass through a "demo payload" boundary before they become README snippets, dashboard cards, issue notes, or module examples.

A safe boundary can be simple:

1. **Start from synthetic data** instead of live output.
2. **Redact risky key names** such as secrets, tokens, passwords, OAuth state, chat IDs, webhook URLs, device IDs, network addresses, and payment fields.
3. **Keep structure visible** so the public repo still teaches how modules exchange data.
4. **Avoid automatic publishing** until the redacted preview is reviewed.
5. **Document limits clearly** because a helper can catch common mistakes, not replace human review.

This keeps examples useful while avoiding claims that the repository contains production telemetry or private assistant state.

## Small actionable task

Add `examples/public_payload_redactor.py`, a dependency-free script that accepts a JSON payload, redacts risky public-demo fields by key name, and prints either JSON or a short Markdown report.

Example safe commands:

```bash
python3 examples/public_payload_redactor.py --format markdown
python3 examples/public_payload_redactor.py --format json
```

## Future module idea

Create a small documentation workflow that asks each new example to include a public payload preview, a redaction pass, and a one-line note explaining which private adapter data must never be committed.

## Security / secret reminder

Do not commit API keys, tokens, OAuth files, webhook URLs, chat IDs, private network addresses, device identifiers, exact home automation state, personal documents, payment details, real logs, or screenshots that reveal private assistant context.
