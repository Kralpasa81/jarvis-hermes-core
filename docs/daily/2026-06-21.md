The emphasis is on a reviewable preview format: useful enough for planning a command center, but clearly synthetic and separated from any private runtime.

## Safe architecture note

Notification workflows should keep three boundaries visible:

1. **Collection boundary** — private adapters may read real sources only inside the private runtime.
2. **Digest boundary** — public examples should use synthetic items and generic categories.
3. **Delivery boundary** — anything sent to Telegram, email, dashboards, or public docs should be redacted and approved when it can affect privacy or external state.

This prevents a demo digest from becoming an accidental chat-log export or a hidden automation trigger.

Do not commit API keys, tokens, OAuth files, chat IDs, webhook URLs, private messages, live device state, private network addresses, payment details, personal documents, screenshots, or exported logs. Public examples should stay synthetic and clearly labeled.
